# Learning paths

Most labs on this site are self-contained: you spin up resources, learn one thing,
and tear them down. **Learning paths are different.** Each path is a guided sequence
of labs built around a single scenario - you carry one app or project forward, adding
a capability at every step, until you reach a complete, production-ready result.

Pick a path below to get started. Each path has its own overview, a step-by-step
tracker, and a single set of resources you keep until you finish.

<LearningPathCatalog />

## How learning paths work

- **One scenario, carried across steps.** A path builds on itself - each step adds to
  the same app and the same resource group instead of starting over.
- **Do the steps in order.** Later steps assume the configuration from earlier ones.
- **Track your progress.** Each path's overview has checkboxes that remember where you
  left off in your browser.
- **Clean up once.** Because everything lives in one resource group, a single delete at
  the end removes it all.

Want just one topic instead of a whole path? The standalone labs in the sidebar cover
each capability - configuration, databases, monitoring, networking, and more - on its
own.
