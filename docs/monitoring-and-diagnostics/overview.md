# Monitoring & Diagnostics

Keep your apps healthy and fast. These hands-on labs show you how to observe
Azure App Service apps with Azure Monitor - collecting telemetry, exploring
logs and metrics, and alerting on problems before your users notice.

## Labs in this section

- **[Monitor your app with Application Insights](./application-insights.md)** -
  Connect a web app to a workspace-based Application Insights resource, enable
  auto-instrumentation, and explore Live Metrics, failures, performance, and
  Logs (KQL), then create a metric alert. Follow the same path three ways
  (Azure Developer CLI, Azure CLI, or the Azure portal) across .NET, Node.js,
  Python, Java, and PHP.

More monitoring and diagnostics labs are on the way, covering log stream and
diagnostic logging, Diagnose and solve problems (AppLens), and advanced console
diagnostics.

:::info App Service Labs complements Microsoft Learn
These labs are hands-on, end-to-end walkthroughs. For reference depth on any
concept, follow the "Learn more" links in each lab to the official
[App Service documentation](https://learn.microsoft.com/azure/app-service/) and
the [Azure Monitor documentation](https://learn.microsoft.com/azure/azure-monitor/).
:::
